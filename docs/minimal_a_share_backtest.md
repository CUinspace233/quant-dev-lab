# A 股日频最小生产化回测 Demo

这个 demo 是一个缩小版量化生产链路：从 raw 数据开始，经过质量检查、清洗、因子计算、组合构建，最后完成一个最小回测。

它不追求策略赚钱，而是帮助你理解量化开发面试中常问的几个核心问题：

- 数据为什么要分 raw 和 processed。
- 因子如何避免未来函数。
- 为什么回测要考虑股票状态、涨跌停和交易成本。
- 从因子到持仓，再到组合净值，中间发生了什么。

## 运行方式

```bash
python examples/minimal_a_share_backtest.py
```

首次运行时，脚本会生成三份模拟 raw 数据：

- `data/raw/trade_calendar.csv`
- `data/raw/daily_quotes.csv`
- `data/raw/stock_status.csv`

之后脚本会读取这些 raw 数据，并生成：

- `data/processed/clean_daily_panel.csv`

raw 文件如果已经存在，脚本不会覆盖。这是为了模拟生产里的原则：原始数据要可追溯，不能随便改。

## 数据层设计

### 交易日历

字段：

- `trade_date`：交易日。
- `is_open`：是否开市。
- `prev_trade_date`：上一个交易日。
- `next_trade_date`：下一个交易日。

真实量化系统不能直接用自然日，因为周末、节假日、临时休市都会影响信号和收益对齐。

### 原始行情

字段：

- `trade_date`
- `symbol`
- `open`
- `high`
- `low`
- `close`
- `volume`
- `amount`
- `adj_factor`
- `limit_up`
- `limit_down`

这里的 `adj_factor` 用来生成复权价格。跨期收益一般不应该直接用原始 `close`，否则分红、送股、拆股会造成价格跳变。

### 股票状态

字段：

- `trade_date`
- `symbol`
- `is_trading`
- `is_st`
- `listed_days`
- `suspend_reason`

这些字段用于构建股票池。A 股回测如果不处理停牌、ST、新股上市天数、涨跌停，结果很容易比真实交易乐观。

## 因子：20 日动量

demo 中的因子是 `momentum_20d`：

> 过去 20 个交易日的前复权累计收益。

核心代码逻辑是：

```python
pct_change(20).shift(1)
```

`pct_change(20)` 计算过去 20 日收益，`shift(1)` 表示 T 日信号只能使用 T-1 及以前的数据。

如果不做 `shift(1)`，就可能把 T 日收盘价用于 T 日决策，相当于回测时提前知道了当天收盘信息，这就是未来函数。

## 策略规则

每 20 个交易日调仓一次：

1. 在可买股票池中筛选股票。
2. 按 `momentum_20d` 从高到低排序。
3. 选择前 5 只。
4. 等权持有。

这是一个非常简单的策略，只用于演示流程。真实策略通常会加入行业约束、风格约束、个股权重上限、换手约束、黑名单等规则。

## 回测口径

demo 使用如下时间顺序：

1. T 日收盘后计算因子和目标持仓。
2. T+1 日使用该持仓计算组合收益。
3. 持仓变化产生换手。
4. 换手乘以交易成本，得到成本扣减。

这样做是为了避免“当天收盘才知道信号，却假设当天已经赚到收益”的时间错位。

## 面试怎么讲

你可以这样介绍这个 demo：

> 我做了一个 A 股日频的最小生产化回测 demo。它不是随机生成价格直接算收益，而是保留了 raw 数据、processed 数据、交易日历、股票状态、复权价格、股票池过滤、因子计算、调仓和交易成本这些真实量化系统里会出现的概念。

如果被问到未来函数：

> 因子计算里我用了 `shift(1)`，确保 T 日信号只使用 T-1 及以前的信息。财报类数据在真实系统里也要按公告日或可获得日对齐，不能按报告期直接使用。

如果被问到为什么还不是实盘系统：

> 这个 demo 只到回测层。真实实盘还需要模拟盘、订单管理、成交回报、持仓同步、风控拦截、监控告警和人工应急流程。

## 真实生产中还需要补什么

这个 demo 有意保持最小，但真实系统还需要：

- 真实交易日历和数据供应商接入。
- 财报公告日、可获得日和数据版本管理。
- 停牌、涨跌停、成交量限制、滑点和冲击成本的更精细建模。
- 行业、市值、风格暴露约束。
- 任务调度、日志、告警和结果归档。
- 模拟盘和实盘交易链路。
